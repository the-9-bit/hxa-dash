## 1. 产品定位

HxA Dash 是 HxA 系列套件之一，定位为 **Agent 团队信息面板**。

核心目标：让人机团队（Human × Agent）的分工、状态、工作流和任务进度实现**可视化**和**量化**。

### 双重价值（#1）

| 视角 | 价值 |
|------|------|
| **对人类** | 员工/工作可视化增加**安全感** — 随时看到 Agent 在做什么、做了什么、接下来做什么。透明度 = 信任。 |
| **对 Agent** | 增加**信息**，实现**有效协作** — Agent 通过面板了解其他成员的状态和工作，做出更好的协作决策。 |

HxA Dash 不只是给人看的监控面板，而是人机团队共享的信息中枢。

### 与 HxA 系列关系

| 套件 | 定位 |
|------|------|
| HxA Connect | Agent 通讯与协作平台 |
| HxA Link | Agent 社交与服务平台 |
| **HxA Dash** | **Agent 团队信息面板** |

HxA Dash 从 HxA Connect 和 GitLab 获取数据，聚合展示。定位是**信息可视化**，不重复任务管理功能。

---

## 2. 目标用户

- **团队管理者**（如 Kevin）：一目了然了解所有 Agent 状态和产出
- **Agent**：获取团队信息，支持协作决策
- **直播/展示场景**：向外部展示人机团队的工作方式和实时状态
- **团队成员（人类）**：了解其他成员（含 Agent）的工作进展

---

## 3. 功能架构（#2 分层可视化）

按层级组织可视化，从宏观到微观：

```
第一层：资源管理（Agent 管理）
  └── 团队全貌：有多少 Agent、各自角色、在线状态、资源分配
第二层：Agent 个体
  └── 单个 Agent 详情：当前任务、历史工作、能力、协作关系
第三层：项目与任务
  └── 工作流看板：项目进展、任务状态、负责人、时间线
```

## 4. 信息流设计（#6）

数据从产生到呈现的完整路径：

```
数据产生                      聚合层                      展示层
─────────                    ──────                      ──────
HxA Connect ──┐
  (在线状态)   │
               ├──→ HxA Dash 后端 ──→  三层面板
GitLab ───────┤     (聚合+持久化)       ├─ 第一层：谁在线？（状态流）
  (MR/Issue)   │     SQLite 落库        ├─ 第二层：他在做什么？（工作流）
               │                        └─ 第三层：项目进展？（任务流）
手动/API ─────┘
  (自定义事件)
```

### 三种信息流

| 信息流 | 数据源 | 触发方式 | 展示位置 |
|--------|--------|----------|----------|
| **状态流** | HxA Connect `/api/bots` | 轮询 30s（P1）→ WebSocket（P2） | 卡片墙在线状态 |
| **工作流** | DB 任务表 + GitLab MR/Issue | API 写入（P1）→ GitLab 自动同步（P2） | 看板 + Agent 详情 |
| **事件流** | GitLab 活动 + 自定义推送 | 轮询 + API 推送 | 时间线 |

### 信息流动规则

1. **状态流**：Agent 在线/离线变化实时反映到卡片墙，颜色 + last_seen 更新
2. **工作流**：任务状态变更（待办→进行中→已完成）同步更新看板和 Agent 详情面板
3. **事件流**：所有活动按时间戳排序汇入时间线，最新在上
4. **联动**：点击卡片 → 筛选该 Agent 的工作流和事件流；点击任务 → 高亮负责 Agent

## 5. 功能需求（#5 多阶段拆解）

### Phase 1 — 骨架版（2026-03-13 直播可用）

目标：先做一个版本找找感觉（#4）。跑通三层可视化的基本框架，数据以配置 + API 实时拉取为主。

| 功能 | 范围 | 说明 |
|------|------|------|
| Agent 卡片墙 | 第一层 | 所有 Agent 一览：名称、角色、在线状态（绿/灰）、当前任务概要 |
| Agent 详情抽屉 | 第二层 | 点击卡片展开：基本信息 + 当前工作列表 + 近期完成工作（从 GitLab 拉取） |
| 任务看板 | 第三层 | 三列（待办/进行中/已完成），手动录入或配置文件导入 |
| 工作时间线 | 全局 | GitLab MR/Issue 活动 + 自定义事件，按时间倒序 |
| 实时刷新 | 全局 | 30 秒自动刷新 + 手动刷新按钮 + 最后更新时间 |
| Basic Auth | 基础 | 统一认证（coco/coco2026） |
| SQLite 持久化 | 基础 | 任务、事件、Agent 配置落库 |

**不含**：协作关系图、量化统计、WebSocket 推送。

---

### Phase 2 — 数据驱动版

目标：减少手动配置，自动从数据源同步；增加 Agent 之间的关联可视化。

| 功能 | 范围 | 说明 |
|------|------|------|
| GitLab 自动同步 | 第三层 | 定时拉取 Issue/MR，自动生成任务卡片和时间线事件 |
| Agent 协作关系图 | 第二层 | 根据项目参与情况自动生成协作网络（谁和谁在同一项目） |
| Agent 历史工作统计 | 第二层 | MR 数量、Issue 关闭数、活跃天数等基础指标 |
| 任务拖拽 | 第三层 | 看板卡片支持拖拽改变状态 |
| WebSocket 实时推送 | 全局 | 替代轮询，实时更新状态变化 |
| 多项目视图 | 第三层 | 按项目（GitLab repo）分组查看任务 |

---

### Phase 3 — 量化洞察版

目标：从可视化进阶到量化分析，支持团队效能评估和趋势追踪。

| 功能 | 范围 | 说明 |
|------|------|------|
| 工作量报表 | 全局 | 按 Agent、项目、时间段统计工作量（MR/Issue/事件） |
| 趋势图表 | 全局 | 团队产出趋势、Agent 活跃度变化曲线 |
| 贡献度排行 | 全局 | Agent 贡献度可视化（不是 KPI 排名，是信息透明） |
| 自定义仪表盘 | 全局 | 用户可拖拽组合不同指标的面板 |
| 导出报告 | 全局 | 导出 PDF/Markdown 团队周报 |

---

### Phase 4 — 平台集成版

目标：与 HxA 生态深度集成，成为 COCO Dashboard 的子模块。

| 功能 | 范围 | 说明 |
|------|------|------|
| COCO Dashboard 嵌入 | 集成 | 作为 #611 的实现，嵌入 coco-dashboard |
| HxA Link 联动 | 集成 | 从 Link 获取 Agent 社交数据 |
| HxA Connect 深度集成 | 集成 | 消息量、响应速度等通讯指标 |
| 多团队支持 | 扩展 | 支持多个独立团队的面板视图 |
| 权限体系 | 扩展 | 角色级权限（管理员/成员/访客） |

### Phase 1 详细需求

#### 4.1 第一层：Agent 资源管理（卡片墙）

团队总览面板，展示所有 Agent 的宏观状态：

| 字段 | 来源 | 说明 |
|------|------|------|
| 名称 | 配置/DB | Agent 名称 |
| 角色 | 配置/DB | 如「全栈工程师」「协调者」 |
| 在线状态 | HxA Connect API | 在线 / 离线 / 最后在线时间 |
| 当前任务 | DB | 正在做什么（简要） |
| 头像 | 配置 | 可选 |

**交互**：
- 卡片按在线状态排序（在线优先）
- 在线状态用颜色指示（绿色在线、灰色离线）
- 点击卡片展开 Agent 详情抽屉

#### 4.2 第二层：Agent 详情抽屉

点击 Agent 卡片展开侧边详情面板（#3 项目透视）：

| 模块 | 说明 |
|------|------|
| 基本信息 | 角色、能力标签、所属团队 |
| 当前工作 | 正在执行的任务列表（从任务看板筛选） |
| 历史工作 | 近期完成的任务 + GitLab MR/Issue 活动 |

Phase 1 不含协作关系图（Phase 2）。

#### 4.3 第三层：项目与任务看板

三列看板：

| 列 | 说明 |
|----|------|
| 待办 | 已分配但未开始的任务 |
| 进行中 | 正在执行的任务 |
| 已完成 | 近期完成的任务 |

每张任务卡片包含：
- 任务标题
- 负责人（Agent 名称）
- 来源（GitLab Issue/MR 链接，可选）
- 状态标签

Phase 1 通过 API 或配置文件录入任务；Phase 2 增加拖拽和自动同步。

#### 4.4 工作时间线

按时间倒序展示近期活动：
- GitLab MR 创建/合并
- GitLab Issue 创建/关闭
- 自定义事件（可通过 API 推送）

每条记录包含：时间戳 + 执行者 + 事件描述

#### 4.5 实时刷新

- 默认每 30 秒自动刷新
- 页面顶部显示「最后更新时间」
- 手动刷新按钮

## 6. 数据来源

### 5.1 HxA Connect API

```
GET /api/bots          → Agent 列表 + 在线状态
GET /api/bots/:name    → 单个 Agent 详情（bio, role, tags, last_seen）
```

认证：Bearer Token（agent_token）

### 5.2 GitLab API

```
GET /api/v4/groups/:id/merge_requests  → MR 活动
GET /api/v4/groups/:id/issues          → Issue 活动
```

认证：PRIVATE-TOKEN

### 5.3 本地数据库（#4 持久化）

SQLite 存储：
- 团队成员配置（角色、头像、标签）
- 任务状态（看板数据）
- 工作时间线（事件记录）
- 协作关系快照

### 5.4 配置文件

`config/team.json` — 团队成员初始配置：

```json
{
  "members": [
    {
      "name": "Jessie",
      "role": "技术负责人 / 协调",
      "avatar": "jessie.png",
      "connect_name": "Jessie",
      "gitlab_username": "Jessie",
      "tags": ["Node.js", "协调", "产品"]
    }
  ]
}
```

## 7. 技术方案

### 6.1 架构

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│   浏览器     │ ←── │  Node.js     │ ←── │ HxA Connect  │
│  (HTML/JS)  │     │  聚合服务    │ ←── │ GitLab API   │
└─────────────┘     │  :3479       │ ←── │ SQLite DB    │
                    └──────────────┘     └──────────────┘
```

### 6.2 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | HTML + CSS + 原生 JS | 轻量、快速、无构建步骤 |
| 后端 | Node.js + Express | 聚合多数据源，提供统一 API |
| 数据库 | SQLite（better-sqlite3） | 轻量持久化，无需额外服务 |
| 部署 | PM2 + Caddy 反向代理 | 与现有基础设施一致 |
| 认证 | Basic Auth | 与 jessie.coco.site 统一（#4） |

### 6.3 API 设计

| 端点 | 说明 |
|------|------|
| `GET /api/team` | 团队成员列表 + 在线状态 |
| `GET /api/team/:name` | 单个 Agent 详情（含历史工作、协作关系） |
| `GET /api/tasks` | 任务看板数据 |
| `POST /api/tasks` | 创建/更新任务 |
| `GET /api/timeline` | 工作时间线 |
| `POST /api/timeline` | 推送自定义事件 |

### 6.4 端口与路由

- 服务端口：3479
- 路由：`jessie.coco.site/hxa-dash/` → `localhost:3479`
- 认证：Basic Auth（coco/coco2026）
