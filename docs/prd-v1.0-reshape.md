# HxA Dash — PRD v1.0 Reshape

> 版本：v1.0 | 日期：2026-03-14 | 基于 Kevin #51 反馈重塑
> 核心转变：从「Agent 监控面板」→「HoA 团队协作面板」

---

## 问题诊断

当前 hxa-dash（v0.9 PRD）的根本问题：

| 现状 | 问题 | 应该是 |
|------|------|--------|
| "Agent 团队信息面板" | 暗示只有 Agent 被观察 | **HoA 团队协作面板** |
| 人只看不操作（D5） | 人是旁观者，不是参与者 | **HoA 都能从面板获得决策信息** |
| 数据展示为主 | 好看但不推动行动 | **每个数据点回答"所以我该做什么？"** |

---

## 三个重塑原则（Kevin #51）

### 1. HxA Friendly — 对 HoA 都有用

**不是**：Human 监控 Agent 的面板
**而是**：Human 和 Agent 共享的团队信息中枢

| 使用者 | 从面板获得什么 |
|--------|--------------|
| **Human（管理者）** | 项目卡在哪？谁空闲可以帮忙？团队整体节奏如何？ |
| **Human（成员）** | 我有哪些 TODO？哪些 MR 等我 review？谁跟我协作最多？ |
| **Agent** | 哪些任务 assigned 给我？哪些依赖我完成才能推进？我的产出趋势如何？ |

> **设计检验**：每个面板功能，问"Agent 看这个有用吗？"如果没用，说明还是在做监控。

### 2. 本质是提升效率

**不是**：好看的数据可视化
**而是**：决策辅助工具

每个数据模块必须回答一个具体问题：

| 模块 | 回答什么问题 | 推动什么行动 |
|------|-------------|-------------|
| Health Score | 这个队友需要帮助吗？ | → 主动介入或重新分配 |
| 卡点检测 | 哪里堵住了？ | → 解除阻塞或调整优先级 |
| TODO 列表 | 我现在该做什么？ | → 直接开始工作 |
| 趋势图 | 团队节奏正常吗？ | → 调整排期或资源 |
| 协作图 | 谁跟谁配合最好？ | → 优化任务分配 |

### 3. 人类想知道卡点和 TODO

**最高优先级的两个新功能**：

**a) 我的视角（My View）**
- 登录/选择身份后，看到**自己相关的**信息
- 我 assigned 的 open issues（我的 TODO）
- 等我 review 的 MR
- 我参与的项目的卡点
- 我的近期产出趋势

**b) 项目卡点（Blockers）**
- 哪些 issue 超过 72h 没有活动？
- 哪些 MR 超过 24h 没人 review？
- 哪些 Agent 连续 4h+ 没有产出？（可能卡住了）
- 哪些任务有依赖但依赖项未完成？

## 产品定位更新

### 旧定位
> HxA Dash 是 Agent 团队信息面板。核心目标：让人机团队的分工、状态、工作流和任务进度实现可视化和量化。

### 新定位
> HxA Dash 是 **HoA 团队协作面板**。核心目标：帮助团队中的每个参与者（Human 和 Agent）了解项目状态、发现卡点、做出更好的协作决策。**不是监控工具，是决策工具。**

### 设计原则更新

| 原则 | v0.9 | v1.0 |
|------|------|------|
| 核心 | Agent 在干啥 + 怎么协作 | **HoA 团队效率——卡点在哪，我该做什么** |
| 用户 | 人看 Agent | **HoA 都是用户** |
| 交互 | 被动消费，人只看不操作 | **决策辅助——看完就知道下一步** |
| 数据 | 可视化为主 | **可行动化（actionable）为主** |
